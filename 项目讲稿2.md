# 项目讲稿（面试用）— 修订版

## 一、项目背景（30秒）

公司有一个穿戴式吸奶器产品 SporraMom W3，产品说明书很长，用户遇到问题时不会翻说明书，要么问客服要么上网搜。但人工客服无法 24 小时在线，网上搜到的信息也不一定准确。所以我们想做一个 AI 客服系统，用户直接问问题，系统自动从说明书里找到答案回复。

---

## 二、系统整体流程（1分钟）

整个系统的核心是 RAG，就是"先找资料，再让 AI 回答"。分五步：

**第一步：数据切分**
把产品说明书切成小块（chunk），每个小块是一个独立的知识点。

**第二步：关键词检索 + 语义检索**
用户问问题后，系统同时用两种方式找相关文档：
- 关键词检索（BM25）：看文档里有没有用户问题中的词
- 语义检索（向量）：理解用户意图，"充不了电"能找到"无法充电"

**第三步：融合排序**
把两种检索结果合并，用 RRF 公式融合排名，再用 Reranker 精排。

**第四步：AI 生成答案**
把检索到的文档塞给大模型，让它基于文档内容生成回答。

**第五步：质量检查**
检查答案的关键词是否在原文中出现，防止 AI 编造信息。

---

## 三、我负责的部分：数据切分（重点讲）

### 3.1 我的工作内容

整个项目里，我负责的是第一步——数据切分。我的任务是把产品说明书切成系统能用的知识块，交付物是一个 JSON 文件。

### 3.2 切分迭代过程

**第一版：按标题自动切**

最初我用代码按说明书的标题自动切分，遇到 `##` `###` 就切一刀。切完发给同伴集成到系统里，跑完测试后反馈回来的结果是：Hit Rate@3 只有 72%。

我看了失败案例，主要问题是安全警告。说明书里 12 条安全警告混在 2 个大块里（769 字 + 548 字），用户问"充电时能用吗"，系统检索到的是整个大块，里面包含"仅限哺乳期""单人专用""禁止睡眠"等一堆无关内容，AI 要从 769 字里找那一句话，找不准。

**第二版：按逻辑单元切**

我改成人工切分，安全警告每条独立，故障排除按类型分，操作步骤按章节分。切完发给同伴，跑完测试反馈回来：Hit Rate@3 提升到 85%，但还有一些失败案例。

我看了失败案例，发现一个问题：用户问"怎么拆"，文档里写的是"分离主机"，两个词意思一样但字不一样，关键词检索匹配不上，语义检索分数也不够高。

**第三版：加口语化注释**

针对这个问题，我在 chunk 里加了同义词注释。比如"怎么拆（拆解、拆卸、取下）：把主机分离"，不管用户用什么词都能匹配上。切完发给同伴，跑完测试反馈回来：Hit Rate@3 提升到 92%。

**第四版：合并高频关注点**

我还发现用户高频问的就是"充多久""电池能用多久"这类问题，就把充电时间、续航、电池容量合并成一个 chunk，用户问任何一个相关问题都能一次性拿到完整信息。

### 3.3 效果验证

我设计了 25 个测试用例，覆盖安全、操作、故障、参数等 6 大类场景。每个用例标注了期望关键词和期望命中 chunk，发给同伴跑评测。

三轮迭代的指标对比：

| 指标 | 第一版（自动切） | 第二版（人工切） | 第三版（加口语化） |
|------|----------------|----------------|------------------|
| Hit Rate@3 | 72% | 85% | 92% |
| MRR | 0.553 | 0.680 | 0.790 |
| 关键词覆盖率 | 81.4% | 86% | 90.9% |

### 3.4 我交付的文件

我交付的是一个 JSON 文件，里面有 4 个字段：

```
{
  "id": 1,
  "topic": "安全警告-充电时禁止使用",
  "content": "充电时禁止佩戴使用。请勿边充电边佩戴...",
  "page": 3
}
```

| 字段 | 作用 |
|------|------|
| id | chunk 编号，方便索引和引用 |
| topic | 主题标签，给人看的，方便调试和评测 |
| content | 正文内容，检索只用这个字段 |
| page | 页码，标记这段内容在说明书的第几页，方便溯源 |

### 3.5 如果是大公司、多产品线

如果公司有很多产品，chunk 文件会更复杂：

```
{
  "id": "chunk_001",
  "content": "...",
  "topic": "...",
  "page": 3,
  "product": "SporraMom W3",
  "source": "产品说明书 v2.1",
  "tags": ["充电", "安全", "禁止"],
  "updated_at": "2025-07-07",
  "version": 1
}
```

这样可以按产品线检索，按标签筛选，按版本追溯。

## 四、我学到了什么

**第一，数据质量比模型更重要。**
花在 chunk 切分上的时间，比调模型参数的收益大得多。切分策略从自动改成人工，命中率直接提升 20%。

**第二，AI 产品必须有评测体系。**
没有量化指标就无法证明效果。25 个测试用例 + 三个指标，让我能客观地说"优化前 72%，优化后 92%"。

**第三，迭代思维。**
第一版有很多问题，但先跑通了完整流程，再根据同伴反馈逐步优化。不是一步到位，而是小步快跑。

**第四，用数据说服人。**
最初按标题自动切效果不好，我用评测数据证明了人工切分更好，说服导师采纳了我的方案。
